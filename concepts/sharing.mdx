# Sharing & Integrations

Roulette provides multiple ways to share your deal flow data with LPs, co-investors, and external tools.

## Shared Links

Create secure, shareable links to give external parties access to selected companies:

```bash
curl -X POST "https://useroulette.com/api/v1/accounts/{account_id}/shared-links" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Q1 2024 Portfolio for ABC Capital",
    "password": "secure-password",
    "expires_at": "2024-06-01T00:00:00Z",
    "companies": [
      { "company_id": "uuid-1", "permissions": "view_deck" },
      { "company_id": "uuid-2", "permissions": "view" }
    ]
  }'
```

### Permission Levels

| Permission | Description |
|------------|-------------|
| `view` | Basic company info only |
| `view_deck` | Company info + pitch deck |
| `full` | All available information |

## Investor Dashboard

Create a dedicated dashboard for your LPs to view portfolio performance:

```bash
# Create an investor dashboard
curl -X POST "https://useroulette.com/api/v1/accounts/{account_id}/investor-dashboard" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "password": "lp-access-2024",
    "shared_permissions": ["view_portfolio", "view_metrics"],
    "is_active": true
  }'
```

### Dashboard Permissions

| Permission | Description |
|------------|-------------|
| `view_portfolio` | List of portfolio companies |
| `view_metrics` | Performance metrics and charts |
| `view_updates` | Company updates and news |
| `download_reports` | Ability to download PDF reports |

## Form Integrations

Connect external forms to automatically create companies in your pipeline.

### Tally Integration

1. Create a Tally form for deal submissions
2. Configure the Tally integration in Roulette:

```bash
curl -X POST "https://useroulette.com/api/v1/accounts/{account_id}/integrations" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "integration_type": "tally",
    "secret": "your-tally-signing-secret"
  }'
```

3. Set your Tally webhook URL to:
   ```
   https://useroulette.com/api/v1/accounts/{account_id}/form-submissions/tally
   ```

### Form Submission Flow

```
┌──────────────┐     ┌──────────────┐     ┌──────────────┐
│  Tally Form  │ ──▶ │   Webhook    │ ──▶ │  Roulette    │
│  Submitted   │     │   Received   │     │  Processes   │
└──────────────┘     └──────────────┘     └──────────────┘
                                                 │
                     ┌───────────────────────────┘
                     ▼
        ┌─────────────────────────────┐
        │  Form Submission Created    │
        │  (optionally converted to   │
        │   company automatically)    │
        └─────────────────────────────┘
```

### Converting Submissions to Companies

Manually convert a form submission to a company:

```bash
curl -X POST "https://useroulette.com/api/v1/accounts/{account_id}/form-submissions/{submission_id}/convert" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "company_status_id": "initial-review-status-uuid"
  }'
```

## Short URLs

Create trackable short URLs for pitch deck links:

```bash
curl -X POST "https://useroulette.com/api/v1/short-urls" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "original_url": "https://example.com/pitch-deck.pdf",
    "account_id": "your-account-id",
    "password": "auto-generate"
  }'
```

Response:
```json
{
  "success": true,
  "data": {
    "short_code": "abc12xyz",
    "short_url": "https://useroulette.com/r/abc12xyz",
    "password": "auto-generated-password"
  }
}
```

## Access Logging

All shared content access is logged for security and analytics:

```bash
# Get access logs for a shared link
curl -X GET "https://useroulette.com/api/v1/accounts/{account_id}/shared-links/{link_id}/access-logs" \
  -H "Authorization: Bearer YOUR_API_KEY"
```

Response:
```json
{
  "success": true,
  "data": {
    "access_logs": [
      {
        "id": "log-uuid",
        "ip_address": "192.168.x.x",
        "user_agent": "Mozilla/5.0...",
        "country": "US",
        "accessed_at": "2024-01-20T15:30:00Z"
      }
    ]
  }
}
```
